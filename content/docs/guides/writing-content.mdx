# Написание контента

Контент пишется в формате MDX (`.mdx`) или Markdown (`.md`) — оба формата поддерживаются.

## Блог-пост

### Файл-пост

Создайте `content/blog/my-post.mdx` или `content/blog/my-post.md`:

```markdown
---
title: "Заголовок"
description: "Описание"
date: "2026-03-07"
tags: ["tag1", "tag2"]
draft: false
---

Содержимое поста...
```

### Директория-пост (с ассетами)

Для постов с картинками создайте папку:

```
content/blog/2025/my-post/
  index.mdx          # контент (или index.md)
  screenshot.png      # ассеты
  diagram.svg
```

В контенте ссылайтесь на ассеты относительными путями:

```markdown
![Скриншот](./screenshot.png)
```

### Frontmatter блога

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок статьи |
| `description` | string | Краткое описание |
| `date` | string | Дата публикации (YYYY-MM-DD) |
| `tags` | string[] | Теги |
| `draft` | boolean | Скрыть из списков (по умолчанию `false`) |
| `geo_block` | object | Гео-блокировка (см. ниже) |

### Мультиязычные посты

Создайте перевод, добавив файл с суффиксом локали:

- `hello-world.mdx` — основной (ru)
- `hello-world.en.mdx` — английская версия

Для директория-постов:

- `index.mdx` — основной
- `index.en.mdx` — перевод

Переключатель языка появляется автоматически, если для поста есть переводы.

### Гео-блокировка

Ограничьте доступ к посту по странам:

```yaml
geo_block:
  countries: ["RU", "DE"]
  message: "Контент недоступен в вашем регионе."
```

Заблокированные посты отображаются с блюром в списках и полной заглушкой на странице поста.

## Документация

Создайте `content/docs/<категория>/my-doc.mdx`:

```markdown
---
title: "Заголовок"
description: "Описание"
order: 1
---

Содержимое документа...
```

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок |
| `description` | string | Описание |
| `order` | number | Порядок в sidebar |

### Индексные страницы категорий

Создайте `index.mdx` в папке категории для страницы-обзора:

```
content/docs/getting-started/
  index.mdx              # Индекс категории → /docs/getting-started
  introduction.mdx       # → /docs/getting-started/introduction
```

## Кастомные страницы

Файл `content/pages/about.mdx` → доступен по `/about`.

```markdown
---
title: "About"
description: "О проекте"
---

Содержимое страницы...
```

Добавьте страницу в навбар через `cloudea.config.ts`:

```ts
navbar: {
  links: [
    { href: "/about", label: { ru: "О проекте", en: "About" } },
  ],
},
```

Чтобы сделать кастомную страницу главной:

```ts
homepage: "about",
```

## Форматирование

MDX/Markdown поддерживает все стандартные возможности:

- **Жирный текст** — `**текст**`
- *Курсив* — `*текст*`
- `Инлайн-код` — `` `код` ``
- ~~Зачеркнутый~~ — `~~текст~~`

## Блоки кода

Тройные обратные кавычки с указанием языка. Кнопка Copy добавляется автоматически:

```tsx
function Welcome({ name }: { name: string }) {
  return <h1>Hello, {name}!</h1>;
}
```

## Callouts (Admonitions)

Используйте компонент `<Callout>` для выделения информации:

```mdx
<Callout type="info">Информационное сообщение</Callout>
<Callout type="tip">Полезный совет</Callout>
<Callout type="warning">Предупреждение</Callout>
<Callout type="danger">Критически важная информация</Callout>
```

<Callout type="info">
Callouts работают только в файлах `.mdx`, так как используют JSX-синтаксис.
</Callout>

## Советы

1. **Используйте заголовки** — они формируют оглавление
2. **Добавляйте код** — делает документацию практичнее
3. **Пишите кратко** — длинные абзацы сложнее воспринимать
4. **Используйте теги** — помогает находить связанные посты
5. **Формат `.md`** — используйте для простого контента без JSX-компонентов
