# Написание контента

Контент на платформе Cloudea Blog пишется в формате MDX — расширении Markdown, которое поддерживает JSX-компоненты.

## Структура файла

Каждый MDX-файл начинается с блока frontmatter:

```markdown
---
title: "Заголовок статьи"
description: "Краткое описание"
date: "2026-03-07"
tags: ["tag1", "tag2"]
---

# Ваш контент здесь
```

### Frontmatter для блога

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок статьи |
| `description` | string | Краткое описание |
| `date` | string | Дата публикации (YYYY-MM-DD) |
| `tags` | string[] | Массив тегов |

### Frontmatter для документации

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок страницы |
| `description` | string | Краткое описание |
| `order` | number | Порядок в навигации |

## Форматирование текста

MDX поддерживает все стандартные возможности Markdown:

- **Жирный текст** выделяется двойными звездочками
- *Курсив* — одинарными
- `Инлайн-код` — обратными кавычками
- ~~Зачеркнутый текст~~ — двойными тильдами

## Блоки кода

Для вставки блока кода используйте тройные обратные кавычки с указанием языка:

```tsx
function Welcome({ name }: { name: string }) {
  return <h1>Hello, {name}!</h1>;
}
```

Платформа автоматически подсвечивает синтаксис для множества языков, включая TypeScript, JavaScript, Python, Bash и другие.

## Списки

Нумерованные списки:

1. Первый пункт
2. Второй пункт
3. Третий пункт

Ненумерованные списки:

- Элемент A
- Элемент B
- Элемент C

## Цитаты

> Цитаты оформляются символом `>` в начале строки. Они полезны для выделения важных замечаний или предупреждений.

## Ссылки и изображения

Ссылки создаются в формате `[текст](url)`, а изображения — `![alt](src)`.

## Советы по написанию

1. **Используйте заголовки** — они формируют оглавление на странице
2. **Добавляйте примеры кода** — они делают документацию практичнее
3. **Пишите кратко** — длинные абзацы сложнее воспринимать
4. **Проверяйте ссылки** — убедитесь, что все ссылки работают
