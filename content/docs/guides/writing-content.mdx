# Написание контента

Контент пишется в формате MDX — расширении Markdown с поддержкой JSX-компонентов.

## Блог-пост

### Файл-пост

Создайте `content/blog/my-post.mdx`:

```markdown
---
title: "Заголовок"
description: "Описание"
date: "2026-03-07"
tags: ["tag1", "tag2"]
draft: false
---

Содержимое поста...
```

### Директория-пост (с ассетами)

Для постов с картинками создайте папку:

```
content/blog/2025/my-post/
  index.mdx          # контент
  screenshot.png      # ассеты
  diagram.svg
```

В MDX ссылайтесь на ассеты относительными путями:

```markdown
![Скриншот](./screenshot.png)
```

### Frontmatter блога

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок статьи |
| `description` | string | Краткое описание |
| `date` | string | Дата публикации (YYYY-MM-DD) |
| `tags` | string[] | Теги |
| `draft` | boolean | Скрыть из списков (по умолчанию `false`) |
| `geo_block` | object | Гео-блокировка (см. ниже) |

### Гео-блокировка

Ограничьте доступ к посту по странам:

```yaml
geo_block:
  countries: ["RU", "DE"]
  message: "Контент недоступен в вашем регионе."
```

Заблокированные посты отображаются с блюром в списках и полной заглушкой на странице поста.

## Документация

Создайте `content/docs/<категория>/my-doc.mdx`:

```markdown
---
title: "Заголовок"
description: "Описание"
order: 1
---

Содержимое документа...
```

| Поле | Тип | Описание |
|---|---|---|
| `title` | string | Заголовок |
| `description` | string | Описание |
| `order` | number | Порядок в sidebar |

Категория определяется именем папки. Поле `order` задаёт сортировку в навигации.

## Кастомные страницы

Файл `content/pages/about.mdx` → доступен по `/about`.

```markdown
---
title: "About"
description: "О проекте"
---

Содержимое страницы...
```

Добавьте страницу в навбар через `cloudea.config.ts`:

```ts
navbar: {
  links: [
    { href: "/about", label: { ru: "О проекте", en: "About" } },
  ],
},
```

Чтобы сделать кастомную страницу главной:

```ts
homepage: "about",
```

## Форматирование

MDX поддерживает все возможности Markdown:

- **Жирный текст** — `**текст**`
- *Курсив* — `*текст*`
- `Инлайн-код` — `` `код` ``
- ~~Зачеркнутый~~ — `~~текст~~`

## Блоки кода

Тройные обратные кавычки с указанием языка. Кнопка Copy добавляется автоматически:

```tsx
function Welcome({ name }: { name: string }) {
  return <h1>Hello, {name}!</h1>;
}
```

## Советы

1. **Используйте заголовки** — они формируют оглавление
2. **Добавляйте код** — делает документацию практичнее
3. **Пишите кратко** — длинные абзацы сложнее воспринимать
4. **Используйте теги** — помогает находить связанные посты
