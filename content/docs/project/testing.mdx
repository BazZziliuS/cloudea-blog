# Тестирование

## Unit-тесты (Vitest)

Запуск:

```bash
npm run test          # однократный запуск
npm run test:watch    # watch-режим
```

Тестируются модули:

| Файл теста | Что проверяет |
|---|---|
| `__tests__/content.test.ts` | Загрузка постов, документации, пагинация, теги, поиск, кэширование |
| `__tests__/config.test.ts` | Конфигурация, навигация, SEO-утилиты, JSON-LD |
| `__tests__/i18n.test.ts` | Словари, совпадение ключей между локалями, fallback |
| `__tests__/rate-limit.test.ts` | Rate limiter, извлечение IP |

## E2E-тесты (Playwright)

Запуск:

```bash
npm run test:e2e
```

Сценарии (`e2e/navigation.spec.ts`):

- Загрузка главной страницы
- Навигация по блогу и документации
- 404 для несуществующих маршрутов
- Открытие/закрытие поиска через Ctrl+K и Escape
- Наличие skip-to-content ссылки и `#main-content`
- Aria-атрибуты (toggle theme)
- Установка cookie локали middleware

Перед первым запуском установите браузеры:

```bash
npx playwright install
```

## Валидация контента

Проверка frontmatter всех постов и документов:

```bash
npx tsx scripts/validate-content.ts
```

Проверяет:

- Обязательные поля (`title`, `date` для постов)
- Формат даты (валидный Date)
- Типы полей (tags — массив, order — число)

При критических ошибках (отсутствует title или date) скрипт завершается с кодом 1 — можно интегрировать в CI.
